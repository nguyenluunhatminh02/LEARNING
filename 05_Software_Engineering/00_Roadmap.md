# 📚 Software Engineering Excellence — Roadmap

## Tổng quan
Kỹ năng kỹ thuật sâu cần thiết cho Staff Engineer / Principal Engineer.
**12 bài học** chia thành **4 phần**.

---

## 📁 01_Code_Quality (Bài 01–03)
| Bài | Tiêu đề | Nội dung chính |
|-----|---------|----------------|
| 01 | Clean Code & SOLID | Naming, functions, SOLID principles, code smells |
| 02 | Design Patterns | Creational, Structural, Behavioral patterns |
| 03 | Testing Strategy | Unit, Integration, E2E, TDD, test pyramid |

## 📁 02_Architecture (Bài 04–06)
| Bài | Tiêu đề | Nội dung chính |
|-----|---------|----------------|
| 04 | Clean Architecture & DDD | Layers, hexagonal, domain modeling, bounded context |
| 05 | Event-Driven Architecture | Event Sourcing, CQRS, Saga, choreography vs orchestration |
| 06 | API Design Mastery | RESTful maturity, versioning, pagination, idempotency |

## 📁 03_DevOps_Practices (Bài 07–09)
| Bài | Tiêu đề | Nội dung chính |
|-----|---------|----------------|
| 07 | Git Mastery | Branching strategies, rebasing, monorepo, conventional commits |
| 08 | CI/CD Pipeline Design | Pipeline stages, deployment strategies, feature flags |
| 09 | Infrastructure & Observability | IaC, monitoring, alerting, incident response |

## 📁 04_Staff_Engineer_Skills (Bài 10–12)
| Bài | Tiêu đề | Nội dung chính |
|-----|---------|----------------|
| 10 | Technical Decision Making | ADR, trade-off analysis, RFC process |
| 11 | Code Review & Technical Debt | Review practices, refactoring strategy, tech debt management |
| 12 | System Thinking & Performance | Performance profiling, capacity planning, reliability engineering |
